# Dark Factories are compiled as LangGraph subgraph nodes

Each Dark Factory is integrated into the Program Layer graph as a compiled LangGraph subgraph, not as an external HTTP service or queue consumer. This means the factory and the orchestrator share a process and the factory receives its `ProjectScope` via shared state rather than a network call. We chose tight coupling here because it keeps the Delivery Run's execution model simple (one graph, one checkpointer, one trace), avoids network failure modes during development, and allows stubs to be swapped in at compile time. The trade-off is that upgrading a real factory requires recompiling the program layer. When the real factories exist and need independent deployment, this decision should be revisited in favour of a message-based integration.
